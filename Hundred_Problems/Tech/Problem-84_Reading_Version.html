Problem 84:
K-Partition Subset Sum

การแบ่งกลุ่มผลรวมเท่ากันด้วยเทคนิค Recursive Backtracking และ Pruning ขั้นสูง

1 หลักการทำงานภาพรวม

ลองจินตนาการว่าคุณมีตัวเลขหลายตัว และมี "กล่อง" จำนวน $k$ ใบ หน้าที่ของคุณคือใส่ตัวเลขลงในกล่องให้ทุกใบมี "น้ำหนักรวมเท่ากัน"

กฎกติกา:

  • • ห้ามเหลือตัวเลขไว้ข้างนอก
  • • ทุกกลุ่มต้องไม่เป็นค่าว่าง
  • • ผลรวมแต่ละกลุ่มต้องเป๊ะ!

จำลองการแบ่ง (nums = [4,3,2,1], k=2, target=5)

กล่องที่ 1
4 1
กล่องที่ 2
3 2

✅ แบ่งสำเร็จ: ทุกกล่องรวมได้ 5

2 เจาะลึกกลไกในโค้ด

🚀 ประสิทธิภาพสูงสุด: Descending Sort

nums.sort(reverse=True)

นี่คือหัวใจของความเร็ว! การลองหยิบ เลขใหญ่ ก่อนจะทำให้เราเจอกรณีที่ "บวกแล้วเกิน" ได้เร็วขึ้นมาก ทำให้โปรแกรมตัดทางเลือกที่ไม่ใช่ทิ้งได้ทันที ไม่ต้องเสียเวลางมเข็มกับเลขตัวเล็กๆ

🔄 การเดินทางย้อนกลับ (Backtracking)

เมื่อสำเร็จ

"บวกเลขจนได้ target แล้วใช่ไหม? ไปหากล่องใบใหม่ต่อเลย!"

เมื่อล้มเหลว

"ไปต่อไม่ได้แล้ว... หยิบเลขตัวล่าสุดออก แล้วลองเลขตัวใหม่ในคิวซิ!"

Performance Analysis

Time Complexity

$O(k \cdot 2^N)$

จำนวนวิธีแบ่งกลุ่มเป็น Exponential แต่ทำงานได้จริงเพราะการตัดกิ่ง (Pruning)

Space Complexity

$O(N)$

ใช้พื้นที่ตามจำนวนตัวเลข ($N$) เพื่อจดบันทึกสถานะและ Recursive Stack

🎯 บทสรุปของโค้ด

เทคนิคที่ใช้ ทำหน้าที่อะไร? ความสำคัญ
Backtracking ลองหยิบเลขใส่กลุ่มทีละสเต็ป ช่วยหาคำตอบที่เป็นไปได้ทั้งหมด
Pruning ตัดทางเลือกที่ไม่ใช่ทิ้งทันที เร่งความเร็วโปรแกรมให้ทันใจ
Used Array จดจำเลขที่ถูกหยิบไปแล้ว ป้องกันความผิดพลาดและการวนซ้ำ
