📝 อธิบายโค้ด ทีละบรรทัด

1️⃣ บรรทัดแรก: ตรวจสอบปัญหา

if not grid or not grid[0]: 
    return 0

ความหมาย:

• if not grid = ถ้า grid ว่าง (ไม่มีข้อมูล)

• or not grid[0] = หรือ grid[0] ว่าง (ไม่มีแถวแรก)

ถ้าเป็นเช่นนั้น คืนค่า 0 (ไม่มีอะไรต้องคำนวณ)

เหตุผล: ถ้าไม่มี grid ก็ไม่สามารถหาเส้นทางได้ ป้องกันโปรแกรม crash

2️⃣ เตรียมข้อมูล: นับแถวและหลัก

rows = len(grid)      # นับแถว
cols = len(grid[0])   # นับหลัก (จำนวนช่องในแถวแรก)

ตัวอย่าง: rows = 3 (มี 3 แถว), cols = 3 (มี 3 หลัก)

3️⃣ สร้าง "บัญชี" หรือ "ตาราง"

dp = [[0 for _ in range(cols)] for _ in range(rows)]

สร้างตาราง 2 มิติมิติวางไว้ (เก็บเลข 0) ขนาด 3x3

ทำไมต้องทำ?

dp[r][c] = ราคาที่ถูกที่สุดที่จะไปถึงช่อง (r,c) เอาไว้เก็บผลลัพธ์ของเส้นทางที่ดีที่สุด

4️⃣ กำหนดจุดเริ่มต้น

dp[0][0] = grid[0][0]

ช่องแรก (0,0) ต้องเสีย = เท่ากับค่าของช่องแรก
ถ้า grid[0][0] = 1 แล้ว dp[0][0] = 1

5️⃣ คำนวณแถวแรก (Row 0)

for c in range(1, cols):
    dp[0][c] = dp[0][c-1] + grid[0][c]

ความหมาย: แถวแรก เดินได้แค่ขวาเท่านั้น

• c-1 = ช่องด้านซ้าย

• dp[0][c-1] + grid[0][c] = ราคาสะสมของด้านซ้าย + ราคาช่องนี้

6️⃣ คำนวณหลักแรก (Col 0)

for r in range(1, rows):
    dp[r][0] = dp[r-1][0] + grid[r][0]

ความหมาย: หลักแรก เดินได้แค่ลงเท่านั้น

• r-1 = ช่องด้านบน

• dp[r-1][0] + grid[r][0] = ราคาสะสมของด้านบน + ราคาช่องนี้

7️⃣ คำนวณส่วนที่เหลือ (ส่วนที่สำคัญที่สุด!) 

for r in range(1, rows):
    for c in range(1, cols):
        dp[r][c] = min(dp[r-1][c], dp[r][c-1]) + grid[r][c]

นี่คือส่วนที่สำคัญที่สุด! ความหมาย: ทุกช่อง เดินได้ 2 ทาง: จากบน หรือ จากซ้าย

1. ราคาจากบน = dp[r-1][c]
2. ราคาจากซ้าย = dp[r][c-1]

หลักการทำงาน:

• min(...) = เลือกทางที่ถูกกว่า

• + grid[r][c] = บวกราคาของช่องปัจจุบันเข้าไป

ทำไมต้อง min()? เพราะเราต้องการเส้นทางที่ ราคาน้อยที่สุด ไม่ใช่มากที่สุด

8️⃣ ส่วนสุดท้าย: คืนคำตอบ

return dp[rows-1][cols-1]

• rows-1 = แถวสุดท้าย (ถ้า 3 แถว คือ แถว 2)

• cols-1 = หลักสุดท้าย (ถ้า 3 หลัก คือ หลัก 2)

• dp[rows-1][cols-1] = ราคาสะสมของจุดหมายปลายทาง

❓ FAQ

Q: ทำไมต้อง range(1, cols) ไม่ใช่ range(0, cols)?

A: เพราะ [0][0] เรากำหนดค่าเริ่มต้นไปแล้ว ถ้าเริ่มจาก 0 จะไปเขียนทับค่าจริงจนผิดพลาดครับ

Q: ทำไมถึงใช้ min() ไม่ใช้ max()?

A: เพราะโจทย์ต้องการหาเส้นทางที่เสีย "น้อยที่สุด" (Min) ถ้าใช้ max จะได้ทางที่แพงที่สุดแทน

Q: ทำไมคำตอบต้องเป็น rows-1 และ cols-1?

A: ภาษาโปรแกรมเริ่มนับตำแหน่งจาก 0 ครับ ถ้ามี 3 แถว ตำแหน่งสุดท้ายคือ 2 (3-1) นั่นเอง
