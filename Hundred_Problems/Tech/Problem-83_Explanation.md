## 🎯 1. หัวใจสำคัญ: ทำไมถึงใช้แค่ `dp[0] * cols`?

ในวิธีปกติ (2D Array) เราใช้พื้นที่ $O(N \times M)$ เพื่อเก็บคำตอบของ **ทุกช่อง** ในแผนที่ แต่ความลับคือ: **"ในการคำนวณช่องปัจจุบัน เราใช้ข้อมูลแค่ 2 จุดเท่านั้น"** คือ:
1.  **จากข้างบน (Top):** ซึ่งอยู่ในแถวก่อนหน้า ($r-1$)
2.  **จากข้างซ้าย (Left):** ซึ่งอยู่ในแถวปัจจุบัน ($r$) ที่เพิ่งคำนวณเสร็จ

ด้วยเหตุนี้ เราไม่จำเป็นต้องเก็บแถวที่ 0, 1, 2... ไว้ทั้งหมด เราแค่ต้องการ **"สมุดโน้ตหน้าเดียว"** ที่คอยอัปเดตค่าใหม่ทับค่าเก่าไปเรื่อยๆ พื้นที่จึงลดลงเหลือเพียง $O(M)$ หรือแค่แถวเดียว!

## 🔍 2. อธิบายโค้ดทีละบรรทัด (The Overwriting Logic)

```python
dp = [0] * cols
dp[0] = grid[0][0]
```
- **Reasoning:** สร้างแถวเปล่าๆ 1 แถว และกำหนดค่าจุดเริ่มต้น

```python
for c in range(1, cols): 
    dp[c] = dp[c-1] + grid[0][c]
```
- **Trace:** ตอนนี้ `dp` เก็บค่าของ **แถวที่ 0** ทั้งแถว (เช่น `[1, 4, 5]`)

```python
for r in range(1, rows):
    dp[0] += grid[r][0]  # อัปเดตช่องซ้ายสุดของแถวใหม่
    for c in range(1, cols):
        # จังหวะสำคัญ! 
        dp[c] = min(dp[c], dp[c-1]) + grid[r][c]
```
- **The "Magic" Explained:**
    - `dp[c]` (ตัวเดิม): ยังเป็นค่าของ **แถวเก่า (ข้างบน)** เพราะเรายังไม่ได้เขียนทับ
    - `dp[c-1]`: เป็นค่าของ **แถวปัจจุบัน (ข้างซ้าย)** เพราะเราเพิ่งคำนวณและเขียนทับไปเมื่อกี้
    - **สรุป:** `min(dp[c], dp[c-1])` ก็คือ `min(Top, Left)` นั่นเอง!

## 🎬 3. การไหลของข้อมูลแบบละเอียด (Step-by-Step Data Stream)

สมมติ `grid = [[1, 3, 1], [1, 5, 1], [4, 2, 1]]`

### 🟢 รอบที่ 0: คำนวณแถวแรก (Base Case)
| Index | 0 | 1 | 2 | สรุปสถานะ `dp` |
| :--- | :--- | :--- | :--- | :--- |
| **Value** | 1 | 4 | 5 | `[1, 4, 5]` |
*หมายเหตุ: นี่คือจุดเริ่มต้นสำหรับใช้เป็น "ค่าบน" ของแถวถัดไป*

### 🔵 รอบที่ 1: คำนวณแถวที่ 1 (Overwriting Row 0)
1. **เริ่มแถว (c=0):** `dp[0] = 1 + 1 = 2` -> `dp` กลายเป็น `[2, 4, 5]`
2. **ช่องกลาง (c=1):** 
   - เทียบ `min(dp[1], dp[0])` -> `min(4, 2)`
   - `dp[1] = 2 + 5 = 7` -> `dp` กลายเป็น `[2, 7, 5]`
3. **ช่องขวา (c=2):**
   - เทียบ `min(dp[2], dp[1])` -> `min(5, 7)`
   - `dp[2] = 5 + 1 = 6` -> `dp` กลายเป็น `[2, 7, 6]`

### 🟠 รอบที่ 2: คำนวณแถวที่ 2 (Overwriting Row 1)
1. **เริ่มแถว (c=0):** `dp[0] = 2 + 4 = 6` -> `dp` กลายเป็น `[6, 7, 6]`
2. **ช่องกลาง (c=1):** 
   - เทียบ `min(dp[1], dp[0])` -> `min(7, 6)`
   - `dp[1] = 6 + 2 = 8` -> `dp` กลายเป็น `[6, 8, 6]`
3. **ช่องสุดท้าย (c=2):**
   - เทียบ `min(dp[2], dp[1])` -> `min(6, 8)`
   - `dp[2] = 6 + 1 = 7` -> `dp` กลายเป็น `[6, 8, 7]`

## 🧠 4. ทำไมวิธีนี้ถึง "ฉลาด" กว่า?

### 1. การประหยัดพื้นที่ (Space Complexity)
- **2D:** $O(N \times M)$ -> ถ้าตาราง $10,000 \times 10,000$ ต้องใช้พื้นที่ **100 ล้านช่อง** (ประมาณ 400MB)
- **1D:** $O(M)$ -> ใช้พื้นที่แค่ **10,000 ช่อง** (ประมาณ 40KB) **ลดลง 2,500 เท่า!**

### 2. ความเร็วระดับ Hardware (Cache Locality)
CPU ชอบอ่านข้อมูลที่อยู่ติดกันในหน่วยความจำ การใช้ 1D Array ทำให้ข้อมูลทั้งหมดวางเรียงตัวกันเป็นเส้นเดียว ทำให้ CPU ดึงข้อมูลเข้า Cache ได้เร็วกว่าการกระโดดไปมาใน 2D Array

## 🎯 สรุปตารางเปรียบเทียบความหมายของตัวแปร

| ตัวแปรในสูตร | ความหมายใน 2D | ความหมายใน 1D (ก่อนเขียนทับ) | บทบาท |
| :--- | :--- | :--- | :--- |
| **Top** | `dp[r-1][c]` | `dp[c]` | ค่าที่ดีที่สุดจากแถวเหนือหัว |
| **Left** | `dp[r][c-1]` | `dp[c-1]` | ค่าที่ดีที่สุดจากช่องซ้ายมือ |
| **Current** | `dp[r][c]` | `dp[c]` (หลังเขียนทับ) | ค่าใหม่ที่กำลังจะบันทึก |
